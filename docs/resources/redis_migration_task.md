## Example

```terraform
terraform {
  required_providers {
    ctyun = {
      source = "ctyun-it/ctyun"
    }
  }
}

# 可参考index.md，在环境变量中配置ak、sk、资源池ID、可用区名称
provider "ctyun" {
  env = "prod"
}


variable "password" {
  type      = string
  sensitive = true
}


resource "ctyun_redis_migration_task" "test" {
  sync_mode     = 1
  conflict_mode = 2
  source_db_info = {
    spu_inst_id  = "425c9173f98b4646a72ce0b986af00b0"
    ip_addr      = "192.168.0.10"
    account_name = "testUser"
    password     = var.password
  }
  target_db_info = {
    spu_inst_id  = "d052e0d8e0204466b2e2673c34247b54"
    ip_addr      = "192.168.0.11"
    account_name = "testUser2"
    password     = var.password
  }

}
```

### Optional

- `conflict_mode` (Number) 数据冲突时的处理办法 1：中断迁移 2：跳过目标key，继续执行 3：覆盖目标key，继续执行
- `operate_type` (Number) 操作类型，可选值：2：结束运行中的任务，3：删除成功或者失败的任务记录 支持更新
- `region_id` (String) 资源池ID，如果不填则默认使用provider ctyun中的region_id或环境变量中的CTYUN_REGION_ID
- `sync_mode` (Number) 同步模式 1：全量同步+增量同步 2：全量同步

### Read-Only

- `complete_time` (String) 结束时间（秒，-1表示时间未知）
- `create_time` (String) 创建时间(秒)
- `id` (String) 任务ID
- `status` (Number) 任务状态 0：初始态 1：运行中 2：成功 3：失败
