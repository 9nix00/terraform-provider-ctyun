## Schema

### Required

- `instance_id` (String) rabbitMq实例ID
- `name` (String) 队列名称
- `vhost` (String) vhost名称

### Optional

- `auto_delete` (Boolean) 是否自动删除，默认false
- `durable` (Boolean) 是否持久化，默认false
- `region_id` (String) 资源池ID，如果不填则默认使用provider ctyun中的region_id或环境变量中的CTYUN_REGION_ID
- `x_dead_letter_exchange` (String) 死信交换器名称。消息被拒绝或过期时将重新发布到该交换器
- `x_dead_letter_routing_key` (String) 死信路由键。死信交换器会发送死信消息到绑定对应路由键的队列上
- `x_expires` (Number) 队列过期时间，过期后队列自动删除，单位为ms
- `x_max_length` (Number) 队列最大消息长度
- `x_max_length_bytes` (Number) 队列消息的总字节数上限
- `x_max_priority` (Number) 队列最大优先级，范围为0~255，默认0
- `x_message_ttl` (Number) 消息过期时间，发布到队列的消息在被丢弃之前可以存活的时间，单位为ms
- `x_overflow` (String) 队列消息处理策略。可选值：drop-head（默认，丢弃最早消息）、reject-publish（拒绝接收新消息）
- `x_queue_mode` (String) 队列模式。可选值：default（默认模式）、lazy（惰性模式）
